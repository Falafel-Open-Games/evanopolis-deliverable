EVANOPOLIS – CORE GAME RULES (V3)

--------------------------------------------------
CORE DESIGN PRINCIPLES
--------------------------------------------------

- Minimalistic: easy to learn, fast to play
- Visual clarity over complexity
- Physical (diegetic) feedback preferred over UI overlays
- Short sessions (target: 10–20 minutes)
- Strategy + randomness (dice-driven movement, light engine building)

--------------------------------------------------
GAME OBJECTIVE
--------------------------------------------------

- A player wins by either:
  - reaching 20 BITCOIN (₿), or
  - being the last active player remaining after all other players have been eliminated

--------------------------------------------------
CORE RESOURCES
--------------------------------------------------

There are 3 core resources:

1. FIAT ($)
2. ENERGY (⚡)
3. BITCOIN (₿) ← victory resource

--------------------------------------------------
RESOURCE FLOW (CORE LOOP)
--------------------------------------------------

FIAT → used to BUY tiles  
TILES → generate ENERGY  
ENERGY → allocated between:
    - SELL → generates FIAT
    - MINE → generates BITCOIN  

Simplified pipeline:

FIAT → buy → ENERGY → (SELL or MINE) → FIAT or BITCOIN

--------------------------------------------------
BOARD STRUCTURE
--------------------------------------------------

- Hexagonal ring composed of 6 sides
- Each side = 1 city / energy type
- Each side has 3 tiles (total: 18 tiles)

Cities / Energy Types:

1. Irkutsk → Cold Geothermal
2. Patagonia → Wind
3. Ciudad del Este → Hydro
4. El Salvador → Geothermal (Volcano)
5. Angra dos Reis → Nuclear
6. Atacama → Solar

Hex ring tile order:

- 0-2: Irkutsk
- 3-5: Patagonia
- 6-8: Ciudad del Este
- 9-11: El Salvador
- 12-14: Angra dos Reis
- 15-17: Atacama

The ring advances in that city order around the board. Tile naming in the
server should always follow these six canonical region names, not the legacy
placeholders from the old board prototype.

--------------------------------------------------
TURN STRUCTURE
--------------------------------------------------

Each turn follows this sequence:

1. Roll dice
2. Move pawn
3. Resolve landing tile:
   - Unowned → option to BUY
   - Owned by another player → PAY toll
   - Owned by self → no payment, then end turn
4. End turn:
   - ENERGY allocation is applied
   - FIAT and BITCOIN are generated

TURN ORDER
--------------------------------------------------

- The first player in a match is the lowest occupied seat index.
- Turn order then proceeds by seat index in ascending order among active players.
- After the last active player, turn order wraps back to the first active player.
- In the current implementation, this means the match starts with player index 0
  and then cycles through the active seated players in order.

--------------------------------------------------
STARTING POSITIONS
--------------------------------------------------

- Pawn starting positions are authoritative server state.
- The gameplay client must not derive opening pawn positions from local scene
  markers or local color assumptions once gameplay has started.
- In the current implementation, a joining player is assigned an initial tile by
  `color_id` before the first turn begins.

Current authoritative opening mapping:

- color `0` starts on tile `12`
- color `1` starts on tile `15`
- color `2` starts on tile `0`
- color `3` starts on tile `3`
- color `4` starts on tile `6`
- color `5` starts on tile `9`

--------------------------------------------------
TILE STATES
--------------------------------------------------

Each tile has 2 possible states:

1. UNOWNED
   - Single layer (height = 1)
   - Neutral base

2. OWNED
   - Double layer (height = 2)
   - Top layer = original tile (city color)
   - Bottom layer = player color (ownership)

--------------------------------------------------
OWNERSHIP VISUAL SYSTEM
--------------------------------------------------

- Ownership is represented physically via stacking
- When a player buys a tile:
  → a second tile (player-colored) is added BELOW

Visual meaning:

- Flat tile → unowned
- Raised tile → owned
- Bottom color → owner

--------------------------------------------------
PLAYER IDENTITY
--------------------------------------------------

- Each player selects a UNIQUE avatar in the lobby
- Each player has a UNIQUE color

Player color is used for:
- Tile ownership base layer
- UI highlights

--------------------------------------------------
TILE INTERACTION
--------------------------------------------------

When landing on a tile:

IF unowned:
  - Player may BUY using FIAT

IF owned by another player:
  - Player first tries to pay the toll in FIAT
  - If the player does not have enough FIAT, the player must instead pay the owner with 1 full BITCOIN
  - If the player does not have 1 full BITCOIN, the toll remains unresolved until that player ends the turn to acknowledge defeat
  - After that acknowledgement, that player is eliminated from the game and all of that player's tiles become unowned again

IF owned by self:
  - No penalty

--------------------------------------------------
ENERGY GENERATION
--------------------------------------------------

- Each owned tile generates ENERGY every turn
- Total ENERGY is the sum of all owned tiles

--------------------------------------------------
ENERGY ALLOCATION SYSTEM
--------------------------------------------------

At the end of each turn, players allocate ENERGY between:

1. SELL (FIAT generation)
2. MINE (BITCOIN generation)

Rules:

- Allocation is expressed as a percentage (0–100%)
- SELL + MINE must always equal 100%
- Allocation persists between turns until changed

Example:

100 ENERGY total

50% SELL → generates FIAT  
50% MINE → generates BITCOIN  

--------------------------------------------------
RESOURCE CONVERSION
--------------------------------------------------

SELL:
- Converts ENERGY into FIAT
- Used for buying tiles and paying tolls when FIAT payment is possible

MINE:
- Converts ENERGY into BITCOIN
- Contributes toward victory

--------------------------------------------------
PLAYER DECISION LOOP
--------------------------------------------------

Each turn includes two key decisions:

1. BOARD DECISION
   - Buy / Pass / Pay
   - If FIAT is insufficient for a toll, transfer 1 full BITCOIN instead

2. ENERGY DECISION
   - Adjust ENERGY allocation between SELL and MINE

--------------------------------------------------
GAME FEEL TARGET
--------------------------------------------------

- Fast decisions
- Visible ownership (no UI dependency)
- Clear "danger vs safe" zones on the board
- Increasing tension as more tiles are owned
- Trade-off between expansion (FIAT) and victory (BITCOIN)

--------------------------------------------------
SCOPE LIMITS (V1)
--------------------------------------------------

Explicitly NOT included:

- No auctions
- No mortgage system
- No jail mechanics
- No complex trading system
- No multi-layer upgrades (max 2 layers per tile)
- No card system (reserved for future)

--------------------------------------------------
FUTURE EXTENSIONS (NOT IN V1)
--------------------------------------------------

- Tile upgrades (3rd layer or structures)
- Card system
- Territory bonuses (own full side)
- Dynamic pricing or scaling costs
- Special event tiles

--------------------------------------------------
SUMMARY
--------------------------------------------------

Core loop:

ROLL → MOVE → BUY / PAY → ALLOCATE ENERGY → PRODUCE → REPEAT

Victory:

- First to 20 BITCOIN (₿), or
- last active player remaining after all other players have been eliminated

Key innovation:

- ENERGY split between FIAT generation and BITCOIN mining
- Ownership represented by STACKED TILES
- Physical, readable, UI-independent system

Elimination:

- A player who cannot resolve a toll with FIAT or 1 full BITCOIN is removed from the game
- That removal happens when the player ends the turn to acknowledge the unpayable toll
- When a player is removed, their owned tiles return to the bank as unowned tiles
- The match continues until only one active player remains, or a player reaches 20 BITCOIN
