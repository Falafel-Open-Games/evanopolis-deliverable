# 004 - Graphical Client Follow-Up Queue

## Status

Approved.

## Purpose

The first three graphical-client milestones have now established the boot,
session-check, and waiting-room flow.

This document is the consolidation point for the remaining follow-up work so
future changes do not get split across `001`-`003`.

## Priority Queue

1. Waiting-room identity metadata and snapshot contract

   Add per-player short name, fixed icon, and fixed color to the authoritative
   waiting-room/player snapshot, then render that data in the waiting-room UI.
   This is the next direct player-facing improvement to the existing lobby.

2. Ready-state undo / unready support

   Allow the local player to cancel a ready state before match start under
   server authority. Keep `rpc_player_ready_state(...)` as the broadcast source
   of truth for the UI.

3. Creator-controlled early match start

   Add a creator-only "start now" action for partially filled rooms once the
   minimum ready count is met. The waiting room should expose creator and
   eligibility state explicitly if this is added.

4. Payment admission tightening at join time

   Make join admission explicitly consult the payment/auth side on the server
   while keeping the Godot handshake unchanged. This is the main server-side
   hardening follow-up from the session-check work.

5. First gameplay root

   Build the first true gameplay scene/root that consumes board state, turn
   lifecycle broadcasts, movement, landing, and reconnect-oriented state.

6. Session refresh / token refresh bridge

   Reuse the boot/session bridge for future token renewal or reconnect flows,
   but only after the current waiting-room path is stable.

## Notes

- The current waiting-room work remains the source of truth for the active
  player flow.
- The next task-specific guide should be written as `005.md` for the top
  priority item here, which is the waiting-room identity setup.
- If a future task becomes broad enough to deserve its own milestone doc,
  create a new devlog rather than expanding this queue into a design dump.
